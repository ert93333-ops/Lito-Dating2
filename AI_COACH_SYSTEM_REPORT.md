# Lito 데이팅 앱 AI 코치 시스템 상세 점검 보고서

Lito 앱의 핵심 차별화 포인트인 **AI 코치 시스템(PRS 및 Coaching Engine)**을 코드 레벨에서 정밀 분석한 결과입니다. 기획과 프롬프트 설계는 매우 훌륭하나, 실제 런칭을 위해서는 몇 가지 치명적인 버그 수정과 연동 작업이 필요합니다.

## 1. 시스템 아키텍처 및 설계 평가

AI 코치 시스템은 단순한 LLM 호출이 아니라, **휴리스틱(규칙 기반) 분석과 LLM 의미 분석이 결합된 하이브리드 엔진**으로 매우 정교하게 설계되어 있습니다.

| 모듈 | 역할 | 평가 |
|---|---|---|
| `prsSignals.ts` | 클라이언트에서 대화 패턴(답장 속도, 질문 빈도, 특정 단어 등)을 수학적으로 추출 | 매우 우수함. 서버 부하를 줄이는 좋은 설계 |
| `prsScoring.ts` | 추출된 패턴과 LLM의 의미 분석 결과를 결합하여 0~100점의 PRS(호감도) 점수 산출 | 정교한 가중치 시스템 적용됨 |
| `prsCoaching.ts` | PRS 점수와 대화 패턴을 기반으로 8가지 코칭 규칙 중 적절한 것을 선택하여 LLM 프롬프트에 주입 | 상황에 맞는 맞춤형 조언 생성 가능 |
| `ai.ts` (API) | 최종적으로 OpenAI API를 호출하여 요약, 톤별 제안, 팁을 JSON 형태로 반환 | 프롬프트 엔지니어링 우수 |

## 2. 치명적인 버그 및 문제점 (런칭 전 필수 수정)

### 2.1. 존재하지 않는 AI 모델 호출 (가장 심각)
- **문제**: `api-server/src/routes/ai.ts`에서 OpenAI API를 호출할 때 `model: "gpt-5.2"` 및 `model: "gpt-4.1-mini"`를 사용하고 있습니다. 이는 존재하지 않는 모델이므로 무조건 에러(`500 Internal Server Error`)가 발생합니다.
- **해결**: 모든 모델명을 `gpt-4o-mini` 또는 `gpt-4o`로 수정해야 합니다.

### 2.2. 가짜 결제(Mock) 연동으로 인한 크레딧 증발
- **문제**: AI 코치는 유료 기능으로 설계되어 크레딧 시스템(`GrowthContext.tsx`)이 구현되어 있습니다. 하지만 크레딧 충전 함수(`buyAiCoachCredits`)가 실제 인앱 결제(RevenueCat 등)와 연동되지 않고 `mockAddConsumable`이라는 가짜 함수를 호출합니다.
- **해결**: 앱스토어 심사 통과 및 실제 수익 창출을 위해 RevenueCat 등 실제 인앱 결제 모듈 연동이 필수적입니다.

### 2.3. 스캠(사기) 감지 로직의 한계
- **문제**: `prsSignals.ts`에 스캠 위험 단어(`"send money"`, `"투자"`, `"코인"` 등)를 감지하는 로직이 있으나, 이는 단순 단어 매칭(Heuristic) 방식이라 우회하기 쉽습니다.
- **해결**: 추후 v2 업데이트 시 LLM을 활용한 문맥 기반 스캠 감지 로직으로 고도화가 필요합니다.

### 2.4. 관리자(Admin) API 보안 취약점
- **문제**: `api-server/src/routes/ai.ts` 하단에 AI 사용량 및 PRS 통계를 볼 수 있는 `/api/admin/prs/aggregates` 등의 엔드포인트가 존재하지만, 인증(Auth) 미들웨어가 전혀 없어 누구나 접근 가능합니다.
- **해결**: 해당 라우트에 관리자 권한을 검증하는 미들웨어(`requireAdmin` 등)를 추가해야 합니다.

## 3. 우수한 점 (유지 및 강조 포인트)

1. **다국어(한/일) 완벽 지원**: 사용자의 UI 언어(`uiLang`)와 상대방의 언어(`targetLang`)를 구분하여, 요약과 팁은 내 언어로, 실제 보낼 메시지 제안은 상대방 언어로 생성하는 로직이 매우 훌륭합니다.
2. **비차단(Non-blocking) UI**: AI 코치 결과가 채팅 입력창을 덮어쓰지 않고, 상단에서 부드럽게 내려오는 팝업(`AiCoachPopup`) 형태로 구현되어 사용자 경험(UX)이 뛰어납니다.
3. **안전장치(Guardrails)**: LLM이 "상대방이 당신을 사랑합니다"와 같이 확정적인 감정을 단언하지 못하도록 프롬프트 레벨에서 강력하게 통제하고 있어, 데이팅 앱으로서의 리스크를 잘 관리하고 있습니다.

## 4. 결론 및 다음 단계
