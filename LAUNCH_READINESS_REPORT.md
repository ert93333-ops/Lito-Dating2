# Lito 데이팅 앱 런칭 전 종합 기능 점검 보고서

본 보고서는 Lito 데이팅 앱의 전체 코드베이스를 분석하여 런칭 전 기능 완성도, 보안 취약점, 미구현 항목을 종합적으로 진단한 결과입니다.

## 1. 핵심 기능 완성도 진단

현재 앱의 주요 기능들은 대부분 구현되어 있으나, 일부 기능은 **프론트엔드 목업(Mock) 상태**이거나 백엔드 연동이 누락되어 있습니다.

### 1.1. 인증 및 소셜 로그인 (완료)
- **구현 상태**: Google, Kakao, LINE, Apple 로그인 모두 서버사이드 OAuth 및 네이티브 연동으로 정상 구현되어 있습니다.
- **보안**: OAuth 콜백 시 `state` 파라미터를 통한 CSRF 방어가 적용되어 있으며, JWT 기반 인증이 정상적으로 작동합니다.

### 1.2. 스와이프 및 매칭 (부분 완료)
- **구현 상태**: `likeUser`, `passUser` API가 구현되어 있으며, 상호 좋아요 시 매칭이 성사되고 채팅방이 생성되는 로직이 정상 작동합니다.
- **문제점**: `AppContext` 내에 `mockConversations`, `mockMatches` 등 하드코딩된 데모 데이터가 여전히 존재하며, 초기 로드 시 이 데이터가 섞여 나올 수 있습니다. 런칭 전 목업 데이터 제거가 필요합니다.

### 1.3. 실시간 채팅 및 번역 (완료)
- **구현 상태**: WebSocket을 통한 실시간 채팅이 구현되어 있으며, 연결 끊김 시 자동 재연결 로직도 포함되어 있습니다.
- **AI 번역**: OpenAI API를 활용한 한/일 양방향 번역 및 발음 기호(가타카나/한글) 제공 기능이 정상 작동합니다.
- **문제점**: 번역 API에서 존재하지 않는 모델인 `gpt-5.2`를 호출하고 있습니다. 이를 `gpt-4o` 또는 `gpt-4o-mini`로 수정해야 합니다.

### 1.4. AI 코치 및 프로필 제안 (부분 완료)
- **구현 상태**: 대화 맥락을 분석하여 코칭 팁을 제공하는 기능과 첫 메시지 제안 기능이 구현되어 있습니다.
- **문제점**: `aiMatching.ts`의 프로필 제안 로직은 실제 AI 모델이 아닌 하드코딩된 휴리스틱 룰(나이, 관심사 교집합 등)에 기반하고 있습니다.

## 2. 수익화 및 결제 (미구현)

가장 시급한 조치가 필요한 부분입니다. 현재 결제 관련 로직은 모두 프론트엔드에서 시뮬레이션만 되고 있습니다.

- **결제 연동 누락**: RevenueCat이나 IAP(In-App Purchase) SDK가 설치되어 있지 않으며, `GrowthContext.tsx`의 `mockUpgradeToPlan` 함수를 통해 가짜 결제만 이루어집니다.
- **조치 필요**: 실제 앱스토어 심사를 통과하고 수익을 창출하려면 RevenueCat 연동 및 백엔드 영수증 검증 로직 구현이 필수적입니다.

## 3. 보안 및 백엔드 취약점

- **Rate Limiting**: AI 관련 엔드포인트(`/api/ai/*`)에는 IP 및 유저 기반 Rate Limit이 적용되어 있으나, 로그인/회원가입 등 인증 엔드포인트에는 Rate Limit이 없어 Brute Force 공격에 취약합니다.
- **보안 헤더**: API 서버에 `helmet` 패키지가 적용되어 있지 않아 기본적인 HTTP 보안 헤더가 누락되어 있습니다.
- **관리자 권한**: `/api/admin/*` 라우트(신고 목록 조회 등)에 관리자 권한 검증 미들웨어가 누락되어 있어, 누구나 접근 가능한 상태입니다.

## 4. 스토어 심사 위험 요소 (Trust & Safety)

- **신원 인증 (Verify ID)**: `verify-id.tsx` 화면의 신원 인증 기능이 실제 KYC 서비스(예: Persona, Sumsub)와 연동되지 않고 `Alert` 창으로만 처리되어 있습니다. 데이팅 앱 심사 시 반려 사유가 될 수 있습니다.
- **신고 기능**: 사용자 신고 기능은 DB에 저장되나, 관리자 알림이나 자동 차단 시스템이 미흡합니다.

## 5. 런칭 전 필수 조치 사항 (Action Items)

1. **AI 모델명 수정**: `src/routes/ai.ts` 파일 내의 `gpt-5.2`를 `gpt-4o-mini`로 일괄 변경
2. **결제 시스템 연동**: RevenueCat SDK 설치 및 `GrowthContext`의 Mock 로직을 실제 결제 로직으로 교체
3. **Mock 데이터 제거**: `AppContext.tsx`에서 `mockData.ts` 의존성 제거 및 초기 상태 빈 배열로 수정
4. **관리자 라우트 보안**: `src/routes/reports.ts` 및 `src/routes/ai.ts`의 `/admin/*` 라우트에 `requireAdmin` 미들웨어 추가
5. **보안 패키지 추가**: API 서버에 `helmet` 및 `express-rate-limit` 적용

위 항목들을 조치한 후 스토어 심사를 진행하는 것을 권장합니다.
