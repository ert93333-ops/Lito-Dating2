# Lito 데이팅 앱 매칭 시스템 점검 보고서

매칭 시스템(스와이프, 좋아요/패스, 매칭 성사, 추천 알고리즘, 대화방 생성)의 전체 흐름을 코드 레벨에서 점검한 결과입니다. 전반적인 뼈대는 잘 갖춰져 있으나, **실제 런칭을 위해서는 하드코딩된 Mock 데이터와 임시 로직들을 걷어내야 합니다.**

## 1. 스와이프 및 추천 (Discover) 로직

### 1.1. 추천 알고리즘 상태
- **현재 구현**: API 서버의 `/api/users/discover` 라우트에서 추천 유저를 반환합니다. 필터링(국가, 나이, 언어 레벨, 관심사)은 정상적으로 동작하며, 한일 양국어 관심사 교차 매칭(예: "여행" ↔ "旅行")도 잘 구현되어 있습니다.
- **문제점**: 실제 DB 유저 외에 `AI_MOCK_USERS`와 `DEMO_USERS`라는 하드코딩된 가짜 유저들이 **항상 추천 풀에 포함**됩니다. 또한, 매칭 호환성 점수(`compatibilityScore`)가 실제 알고리즘이 아닌 `70 + Math.floor(Math.random() * 25)` 형태의 **랜덤 값으로 부여**되고 있습니다.
- **조치 필요**: 런칭 전 `users.ts`에서 `AI_MOCK_USERS`와 `DEMO_USERS`를 제거하고, 호환성 점수 계산 로직을 실제 프로필 기반으로 고도화해야 합니다.

### 1.2. 스와이프 UI
- **현재 구현**: React Native Reanimated와 Gesture Handler를 사용해 60fps의 부드러운 스와이프 애니메이션이 구현되어 있습니다.
- **상태**: 매우 훌륭하며 수정할 부분이 없습니다.

## 2. 좋아요 / 패스 및 매칭 성사 로직

### 2.1. DB 연동 상태
- **현재 구현**: 사용자가 좋아요를 누르면 `/api/users/:id/like` API가 호출되어 DB의 `swipe_likes` 테이블에 기록됩니다. 상대방도 나를 좋아요 한 기록이 있으면 `matches` 테이블에 매칭 기록이 생성됩니다.
- **문제점**: DB 연동 자체는 정상이나, 상대방이 가짜 유저(AI/Demo)일 경우 DB가 아닌 서버 메모리(`guestLikes`, `guestMatches`)에만 저장되는 **인메모리 폴백 로직**이 섞여 있습니다. 서버가 재시작되면 이 기록은 모두 날아갑니다.
- **조치 필요**: 가짜 유저를 제거함과 동시에 인메모리 폴백 로직을 완전히 삭제하고, 오직 DB(`swipe_likes`, `matchesTable`)만 바라보도록 코드를 정리해야 합니다.

## 3. 대화방 (Conversation) 생성 로직

### 3.1. 클라이언트 대화방 상태 관리
- **현재 구현**: 매칭이 성사되면 클라이언트(`AppContext.tsx`)에서 즉시 `conv_{userId}` 형태의 ID로 대화방 객체를 생성하고 채팅 탭에 추가합니다.
- **문제점**: 대화방 목록을 서버에서 불러오는 API(`/api/chat/conversations`)가 존재하지 않습니다. 앱을 껐다 켜면 `mockData.ts`에 하드코딩된 `mockConversations`가 초기값으로 로드되어, **실제 매칭된 대화방은 사라지고 가짜 대화방만 보이게 됩니다.**
- **조치 필요**: 
  1. API 서버에 사용자의 전체 대화방 목록을 내려주는 GET API를 추가해야 합니다.
  2. 클라이언트(`AppContext.tsx`)의 `fetchConversations` 로직을 구현하여 앱 시작 시 서버에서 대화방 목록을 동기화하도록 수정해야 합니다.
  3. `mockConversations` 의존성을 완전히 제거해야 합니다.

## 4. 종합 결론 및 우선순위

현재 매칭 시스템은 **"데모 시연용"**으로는 완벽하게 동작하지만, **"실제 서비스용"**으로는 데이터 영속성 문제가 큽니다.

**런칭 전 필수 조치 사항 (우선순위 순):**
1. **Mock 데이터 제거**: `users.ts`와 `AppContext.tsx`에서 하드코딩된 데모 유저와 대화방 데이터를 완전히 삭제합니다.
2. **대화방 목록 API 구현**: 매칭된 유저들과의 대화방 목록을 DB에서 불러오는 API를 추가하고 클라이언트에 연동합니다.
3. **인메모리 로직 제거**: 좋아요/패스/매칭 기록을 서버 메모리가 아닌 100% PostgreSQL DB에만 의존하도록 수정합니다.
