1. 프로젝트 개요
==================================================

서비스명: LITO
서비스 성격: 한국-일본 크로스보더 성인 관계 형성 앱
핵심 정의:
- LITO는 AI 앱이 아니다.
- LITO는 “언어 장벽과 초기 불안을 낮춰 첫 상호 대화를 안전하게 시작하게 하는 제품”이다.
- 제품의 코어는 번역 가능한 프로필, 상호 매칭 후 무료 1:1 채팅, 신고/차단/삭제까지 포함한 신뢰 구조다.

고정 전제:
- 18세 이상만 가입 가능
- 한국 우선 출시, 서울/수도권 중심 launch focus
- 단, 하드 geofence는 아직 미정
- 공개 피드 없음
- candidate browse/list → mutual like → 1:1 텍스트 채팅 구조
- 상호 매칭 후 기본 텍스트 채팅은 무료
- KR↔JP 프로필 번역과 채팅 번역이 코어
- 신고/차단/연락처 차단/계정 삭제는 코어 기능
- 앱 내 삭제 시작 + 웹 삭제 요청 필수
- 모든 디지털 유료 기능은 스토어 결제 기준만 허용
- 코어 AI는 번역, 코칭은 선택형 보조 기능
- 코칭 자동 실행 금지
- zero_credit이어도 채팅과 번역은 유지
- unsafe_interaction에서는 코칭보다 safety CTA 우선

권장 기술 스택:
- Mobile: Expo / React Native / expo-router
- Backend: Node.js / Express / TypeScript
- DB: PostgreSQL
- ORM: Drizzle 또는 동등 수준 type-safe ORM
- Realtime: WebSocket
- Queue/Cache: Redis
- Storage: Object Storage
- Billing: App Store / Google Play 검증 어댑터
- AI: 외부 번역/LLM provider, 단 기능별 opt-in 및 최소 전송 원칙 준수

# 1. MVP 정의
LITO MVP는 18세 이상 성인만 가입 가능한 한국 우선·서울/수도권 중심 런치 포커스의 KR↔JP 크로스컬처 관계 형성 앱이다. 사용자는 Apple/Google/Email 기반 계정으로 가입하고, 번역 가능한 프로필을 작성한 뒤, 공개 피드 없이 후보 프로필을 탐색하고 상호 호감이 성사되면 1:1 텍스트 채팅을 무료로 시작한다. KR↔JP 프로필 번역과 실시간 채팅 번역이 핵심 가치이며, 최소 신뢰 스택은 18+ 게이트, 계정 인증, 신고/차단, 연락처 차단, 탈퇴 경로, optional verification 확장 구조로 구성된다.

# 2. 제품 원칙
- Trust before scale
- Translation before coaching
- Match-first, 1:1 only
- Free core conversation
- Layered trust stack
- Store-compliant monetization only
- Deletion must be real and reachable
- Culture-aware copy by default

# 3. 핵심 목표
- 가입한 사용자가 한 세션 안에 프로필을 완성하고 탐색에 진입
- KR↔JP 첫 대화 장벽을 번역으로 낮춤
- 신고/차단/연락처 차단/탈퇴를 기본값으로 제공
- 이후 optional verification, AI 코칭, 유료 SKU 확장 가능 구조 확보

# 4. 비목표
- 공개 피드/커뮤니티/타임라인
- 랜덤 채팅/익명 오픈채팅/그룹 채팅
- 음성/영상통화 중심 경험
- AI 의존형 관계 형성
- 기본 텍스트 대화의 유료 잠금
- 정밀 위치 노출 중심 제품

# 5. 핵심 가치
- 이해된다: 프로필을 KR↔JP로 읽을 수 있다
- 시작할 수 있다: 상호 매칭 후 바로 무료 1:1 텍스트 대화
- 덜 무섭다: 신고/차단/연락처 차단/탈퇴가 명확
- 덜 오해된다: 실시간 번역과 원문 확인 구조로 의미 왜곡 최소화

# 6. 핵심 잠금 요약
- 18+ only
- 25~39는 GTM/persona 중심 연령이지 가입 제한 아님
- 한국 우선·서울/수도권 중심 launch focus
- 하드 geofence 여부는 미정
- KR↔JP 프로필 번역 Must
- KR↔JP 실시간 채팅 번역 Must
- AI 코칭은 Should/Later 경계
- 상호 매칭 후 기본 텍스트 채팅 무료
- 공개 피드 없음
- 디지털 유료 기능은 스토어 결제만
- 계정 삭제는 앱 내 시작 + 웹 삭제 요청
- 로그인 목표는 Apple/Google/Email

10. 구현 규칙
==================================================

1. message와 translation은 반드시 분리 구현한다.
2. 번역 실패는 메시지 전송 실패로 처리하면 안 된다.
3. coach debit은 성공 반환 후 원자적으로 처리한다.
4. unsafe_interaction이면 coach 차단, safety CTA 우선이다.
5. consent_not_given이면 외부 AI 호출 금지다.
6. zero_credit이어도 채팅과 번역은 유지한다.
7. delete_request 시작 시 즉시 비노출/세션 종료를 반영한다.
8. contact block은 원문 연락처 대신 hash 기반으로 저장한다.
9. raw prompt/response 장기 저장 금지다.
10. admin tool은 최소권한/감사로그 원칙을 지킨다.
