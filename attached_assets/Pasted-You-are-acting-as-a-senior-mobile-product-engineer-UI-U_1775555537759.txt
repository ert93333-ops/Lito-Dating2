==================================================

Required behavior must be exactly this:

- show translated text
- hide pronunciation text

Pronunciation ON:
- show original text
- show translated text
- show pronunciation text

Pronunciation OFF again:
- show original text
- show translated text
- hide pronunciation text

Critical rule:
The pronunciation toggle must ONLY control pronunciation visibility.
It must NEVER:
- hide translated text
- replace translated text
- collapse the translation area
- swap translation and pronunciation

Use separate data fields:
- originalText
- translatedText
- pronunciationText

Use separate logic/state:
- showTranslation (normally true when translatedText exists)
- showPronunciation (toggle)

Translation visibility must never depend on showPronunciation.

Do NOT use patterns like:
- showPronunciation ? pronunciationText : translatedText
- if pronunciation on, hide translation
- one shared boolean for translation and pronunciation
- one shared render slot for translation and pronunciation

Correct render structure for each message:

MessageBubble
- OriginalText
- TranslatedText   // always visible if translatedText exists
- PronunciationText // only visible when showPronunciation is ON and pronunciationText exists

- it may show/hide pronunciation lines across the chat
- but it must NEVER affect translation visibility
