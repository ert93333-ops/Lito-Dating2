==================================================
MVP IMPLEMENTATION RULES
==================================================

For MVP:
- prefer practical heuristics over overengineered NLP
- create clean abstractions so v2 can improve later
- use current message data, timestamps, sender identity, translation state, and chat metadata if available
- where full semantic understanding is unavailable, create reasonable proxy functions

Examples:
- FollowUpQuestionRate:
  detect partner questions that reference the previous message or continue a known topic

- ContingentReplyScore:
  detect whether partner replies continue the previous semantic thread

- DisclosureBalance:
  estimate whether both sides are sharing at roughly similar depth

- PartnerReinitiation:
  detect whether the partner starts again after a silence gap

- BaselineAdjustedReplySpeed:
  compare recent reply speed against the partner’s own historical baseline

- TopicAlignment:
  semantic or keyword/topic overlap across adjacent turns

- OtherFocusScore:
  partner-focused language, direct reference to the other person, name use, interest cues

- WarmthScore:
  validating, kind, supportive, emotionally warm language

- AuthenticityScore:
  specific, non-generic content vs empty/generic template content

- AvailabilitySharing:
  explicit timing openness such as weekend, after work, specific days

- GenericTemplatePenalty:
  repeated generic expressions, duplicated patterns, low-specificity replies
