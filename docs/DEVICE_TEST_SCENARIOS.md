# LITO 실기기 테스트 시나리오

> 출시 전 마지막 게이트. iPhone + Android 실기기에서 아래 시나리오를 **모두 통과**해야 출시 가능.

## 사전 조건

| 항목 | 요구사항 |
|------|----------|
| API 서버 | Railway/VPS에 배포 완료, HTTPS 접근 가능 |
| DB | PostgreSQL 가동 중, 마이그레이션 완료 |
| GCS | 버킷 생성 + CORS 설정 + 서비스 계정 키 연결 |
| 환경변수 | `EXPO_PUBLIC_DOMAIN`이 프로덕션 API 도메인을 가리킴 |
| 프로덕션 플래그 | `ENABLE_AI_PERSONAS=false`, `ENABLE_DEMO_USERS=false` |
| Expo 빌드 | `eas build --profile preview` 또는 `expo start --dev-client` |

---

## TC-01: 신규 가입 (이메일)

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 앱 실행 | 로그인 화면 표시 |
| 2 | "회원가입" 탭 선택 | 이메일/비밀번호 입력 폼 |
| 3 | 이메일 + 비밀번호 입력 → "가입" | 프로필 설정 화면으로 이동 |
| 4 | 네트워크 에러 시 | 에러 메시지 표시, 앱 크래시 없음 |

**검증**: `POST /api/auth/register` → 201, JWT 토큰 발급

---

## TC-02: 프로필 설정

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 닉네임, 나이, 언어 입력 | 각 필드 정상 입력 |
| 2 | 메인 사진 촬영/선택 | 카메라/갤러리 접근 → 업로드 진행 표시 |
| 3 | 추가 사진 3장 선택 | 각각 업로드 완료 표시 |
| 4 | 업로드 실패 시 | 빨간 경고 오버레이 + "재시도" 버튼 |
| 5 | 관심사 3개 이상 선택 | 선택된 관심사 하이라이트 |
| 6 | "시작하기" 탭 | discover 화면으로 이동 |

**검증**: `PUT /api/auth/profile` body에 `photos[]` 포함, `GET /api/auth/me`에서 photos 반환

## TC-03: Discover (스와이프)

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | discover 탭 진입 | 로딩 스피너 → 실제 유저 카드 표시 |
| 2 | 유저 없을 때 | "주변에 새로운 사람이 없습니다" 빈 상태 |
| 3 | 오른쪽 스와이프 (Like) | 카드 사라짐 + 다음 카드 |
| 4 | 왼쪽 스와이프 (Pass) | 카드 사라짐 + 다음 카드 |
| 5 | 별 버튼 (Super Like) | 잔여 횟수 뱃지 감소, 한도 초과 시 비활성화 |
| 6 | 네트워크 끊김 | 에러 UI + "재시도" 버튼 |
| 7 | 재시도 버튼 탭 | 데이터 재로딩 |

**검증**: `GET /api/users/discover` → 200, AI 유저 미포함 (ENABLE_AI_PERSONAS=false)

## TC-04: 매칭

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 상호 Like 발생 | "매칭되었습니다!" 모달 표시 |
| 2 | matches 탭 진입 | 매칭된 유저 목록 표시 |
| 3 | 매칭 없을 때 | "아직 매칭이 없습니다" 빈 상태 |
| 4 | 매칭 유저 탭 | 채팅 화면으로 이동 |

**검증**: `GET /api/chat/conversations` → 200, 매칭된 대화방 목록 반환

## TC-05: 채팅

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 채팅방 진입 | 기존 메시지 로딩 (있으면) |
| 2 | 메시지 입력 → 전송 | 내 메시지 우측 버블, 전송 완료 표시 |
| 3 | 상대방 메시지 수신 | 좌측 버블, 실시간 (WebSocket) |
| 4 | 읽음 표시 | 내 메시지 옆 '1' → 상대가 읽으면 사라짐 |
| 5 | 긴 메시지 스크롤 | 부드러운 스크롤, 최신 메시지로 자동 이동 |

**검증**: WebSocket 연결 정상, `GET /api/chat/:id/messages` → 200

## TC-06: 백그라운드 복귀

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 앱을 백그라운드로 보냄 (홈 버튼) | — |
| 2 | 30초 대기 | — |
| 3 | 앱 복귀 | WebSocket 재연결 + 데이터 새로고침 |
| 4 | 5분 대기 후 복귀 | 동일하게 재연결 + 새로고침 |
| 5 | 비행기 모드 ON → 앱 복귀 | 에러 상태 표시, 크래시 없음 |
| 6 | 비행기 모드 OFF | 자동 재연결 시도 |

**검증**: AppState 리스너 + 지수 백오프 WebSocket 재연결

## TC-07: 프로필 수정

| 단계 | 동작 | 기대 결과 |
|------|------|-----------|
| 1 | 설정 → 프로필 수정 | 현재 프로필 데이터 로딩 |
| 2 | 사진 추가 (최대 6장) | 업로드 진행 → 완료 |
| 3 | 사진 삭제 (X 버튼) | 즉시 제거, 최소 1장 유지 경고 |
| 4 | 사진 순서 변경 (↑ 버튼) | 순서 변경 반영 |
| 5 | 닉네임/자기소개 수정 | 입력 반영 |
| 6 | "저장" 탭 | 서버 저장 + 이전 화면 복귀 |
| 7 | 다시 프로필 수정 진입 | 저장된 데이터 그대로 표시 (round-trip) |

**검증**: `PUT /api/auth/profile` → photos[] 포함, `GET /api/auth/me` → 동일 데이터

## TC-08: 네트워크 에러 전체

| 화면 | 테스트 | 기대 결과 |
|------|--------|-----------|
| discover | 비행기 모드에서 진입 | 에러 UI + 재시도 버튼 |
| matches | 비행기 모드에서 진입 | 로딩 실패 표시 |
| chat | 비행기 모드에서 메시지 전송 | 전송 실패 표시, 크래시 없음 |
| profile-setup | 업로드 중 네트워크 끊김 | 실패 오버레이 + 재시도 |
| profile-edit | 저장 중 네트워크 끊김 | 에러 메시지, 데이터 손실 없음 |
