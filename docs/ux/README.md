# UX

UX 리서치, 화면 설계, 플로우, 디자인 시스템 참조 저장소.

---

## 폴더 구조

```
ux/
  README.md           — 이 파일
  flows/              — 주요 사용자 플로우 다이어그램 (Mermaid 또는 링크)
  screens/            — 화면별 UX 명세 및 상태 정의
  design_system.md    — 색상, 타이포그래피, 컴포넌트 규칙 참조
  usability_tests/    — 사용성 테스트 결과 기록
```

---

## 핵심 플로우 현황

| 플로우 | 구현 상태 | 문서 상태 |
|---|---|---|
| 온보딩 (언어 선택 → 프로필 설정) | 구현 완료 | 미작성 |
| 탐색 / 스와이프 | 구현 완료 | 미작성 |
| 매칭 후 채팅 | 구현 완료 | 미작성 |
| AI 번역 / 답장 제안 | 구현 완료 | 미작성 |
| 신고 / 차단 | 구현 완료 | 미작성 |
| 설정 / 연락처 차단 | 구현 완료 | 미작성 |

---

## 업데이트 규칙

1. 신규 화면 또는 플로우 추가 시 `flows/` 또는 `screens/` 에 명세를 먼저 작성하고 구현한다.
2. 사용성 테스트 완료 후 72시간 이내에 결과를 `usability_tests/` 에 기록한다.
3. 디자인 시스템 변경(색상, 컴포넌트 규칙 등)은 `design_system.md` 를 먼저 업데이트한 뒤 코드에 반영한다.
4. UX 결정 중 제품 방향에 영향을 주는 것은 `docs/decision_log.md` 에 반영한다.
